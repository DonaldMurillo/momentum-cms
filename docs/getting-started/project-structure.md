# Project Structure

Overview of the files generated by `create-momentum-app`.

## Angular + Express

```
my-app/
├── src/
│   ├── app/
│   │   ├── app.ts                  # Root component
│   │   ├── app.config.ts           # Client providers
│   │   ├── app.config.server.ts    # Server providers (Momentum admin routes)
│   │   ├── app.routes.ts           # Angular router config
│   │   ├── main.ts                 # Client bootstrap
│   │   └── main.server.ts          # Server bootstrap
│   ├── collections/
│   │   └── posts.ts                # Sample collection definition
│   ├── momentum.config.ts          # Momentum CMS configuration
│   ├── server.ts                   # Express server setup
│   └── styles.css                  # Global styles + Tailwind + theme variables
├── .env.example                    # Environment variables template
├── angular.json                    # Angular CLI config
├── package.json
├── tailwind.config.js              # Tailwind with Momentum preset
└── tsconfig.json
```

## Key Files

### `momentum.config.ts`

Central configuration file. Defines your database adapter, collections, auth, storage, seeding, and plugins.

```typescript
import { postgresAdapter } from '@momentumcms/db-drizzle';
import { momentumAuth } from '@momentumcms/auth';
import { Posts } from './collections/posts';

export default {
	db: postgresAdapter({ connectionString: process.env['DATABASE_URL']! }),
	collections: [Posts],
	auth: momentumAuth({
		/* auth options */
	}),
};
```

### `collections/posts.ts`

A sample collection showing the basic pattern. This is where you define your data model.

### `server.ts` (Angular) / `server/middleware/00-init.ts` (Analog)

Server entry point where Momentum is initialized and API routes are registered.

### `styles.css`

Includes Tailwind directives and Momentum theme CSS variables for light and dark modes.
